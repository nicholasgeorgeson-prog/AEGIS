What is AEGIS?

AEGIS is a Flask-based web application that serves as a comprehensive document quality assurance and analysis platform for aerospace technical writing. It provides:

  • Automated Quality Checking: 84 offline quality checkers covering grammar, spelling, style, clarity, compliance, technical requirements, structure, readability, and formatting
  • Requirement Extraction: Automatically extract and categorize requirements, guidance, descriptive text, and informational content from documents
  • Role Analysis: Extract and analyze organizational roles from documents with RACI matrix generation and relationship mapping
  • Document Analytics: Comprehensive metrics, trend analysis, and comparative analytics across your document portfolio
  • Link Validation: Batch hyperlink checking with status tracking and deep browser rescanning for JavaScript-rendered pages
  • Document Comparison: Side-by-side diff view with change highlighting and section-level comparison
  • Statement of Work Generation: Template-based SOW generation from extracted requirements and organizational roles

Installation & Setup

AEGIS provides multiple installation methods to suit different preferences and environments. Choose the method that works best for your organization.

Installation Methods

Windows Installer (Recommended for Windows Users) ‚ñº

The Windows installer provides a one-click installation experience with all dependencies bundled.

1

Download the Installer

Download AEGIS-5.0.0-Setup.exe from the releases page. Verify the file size matches the published checksum.

2

Run the Installer

Double-click the .exe file. Windows may show a security warning about an unrecognized publisher. Click "Run anyway" to proceed.

3

Accept License Agreement

Read and accept the AEGIS license agreement. Choose the installation location (default is C:\Program Files\AEGIS).

4

Install Dependencies

The installer automatically installs required Python packages including Flask, SQLite3, and document processing libraries. This may take 2-5 minutes.

5

Launch AEGIS

Check the "Start AEGIS" checkbox and click "Finish". Your default browser will open to localhost:5050. AEGIS is now ready to use.

üí° Tip

The Windows installer also creates a Start Menu shortcut and a desktop icon for easy access. Use these to launch AEGIS in the future.

Manual Installation (All Platforms) ‚ñº

For advanced users or Linux/macOS, manual installation provides complete control over the installation process.

Prerequisites

  • Python 3.8+ installed and accessible from command line
  • Git (optional but recommended)
  • Terminal/Command Prompt access
1

Download Source Code

Clone the repository or download the source ZIP:

git clone https://github.com/aegis/aegis.git
cd aegis

Or download AEGIS-5.0.0-source.zip and extract to a folder.

2

Create Virtual Environment

Create and activate a Python virtual environment:

# macOS and Linux
python3 -m venv venv
source venv/bin/activate

# Windows
python -m venv venv
venv\Scripts\activate
3

Install Dependencies

Install required Python packages from requirements.txt:

pip install --upgrade pip
pip install -r requirements.txt

This installs Flask, SQLAlchemy, python-docx, PyPDF2, and other dependencies. Installation takes 2-5 minutes depending on your internet speed.

4

Initialize Database

Create the initial database and tables:

python scripts/init_db.py

This creates the SQLite database and initializes all required tables for storing scan results, document metadata, and role information.

5

Start the Server

Launch the Flask development server:

python app.py

You should see output indicating the server is running on http://localhost:5050. Open this URL in your web browser.

Verify Installation

To verify your installation was successful:

  1. Open your web browser to http://localhost:5050
  2. You should see the AEGIS dashboard with metric tiles
  3. Try uploading a test document through the drop zone
  4. Check that the document appears in the Scan History
‚ö†Ô∏è Important

Keep the terminal window open where you ran python app.py. Closing this window will stop the AEGIS server. To stop the server intentionally, press Ctrl+C in the terminal.

Docker Installation ‚ñº

Docker provides a containerized environment that eliminates installation complexity and ensures consistency across different machines.

Prerequisites

  • Docker Desktop installed and running
  • 20 GB available disk space for the container image
1

Pull Docker Image

Pull the AEGIS Docker image from Docker Hub:

docker pull aegis/aegis:5.0.0
2

Create Data Volume

Create a persistent volume for storing scan history and databases:

docker volume create aegis-data
3

Run Container

Start the AEGIS container with volume mounting:

docker run -d \
  --name aegis \
  -p 5050:5050 \
  -v aegis-data:/app/data \
  aegis/aegis:5.0.0
4

Access AEGIS

Open your browser to http://localhost:5050. The container will be ready within 30 seconds of startup.

5

Manage Container

Use Docker commands to manage AEGIS:

# View logs
docker logs aegis

# Stop AEGIS
docker stop aegis

# Start AEGIS again
docker start aegis

# Remove container (keeps data volume intact)
docker rm aegis
üí° Tip

Docker installations are perfect for production deployments, team environments, and CI/CD pipelines. Data persists even when the container is stopped or removed.

üí° Tip: Score Interpretation

    ‚ö†Ô∏è Analysis Takes Too Long

    Troubleshooting & FAQ

    Solutions to common problems and frequently asked questions about AEGIS.

    General Issues

    AEGIS Won't Start / Server Error ‚ñº

    Problem

    AEGIS fails to start or shows "Server Error" on page load.

    Solutions

    1. Check Port 5050: Verify port 5050 is available. If another application uses it, kill the process or change AEGIS port in config.json
    2. Restart Server: Stop AEGIS (Ctrl+C) and run python app.py --debug again
    3. Check Logs: Look in the terminal output for error messages. Common issues: missing dependencies, database corruption
    4. Repair Database: If database is corrupted, delete scan_history.db and AEGIS will create a fresh one on startup (you'll lose scan history)
    5. Clear Cache: In your browser, press Ctrl+Shift+Delete to clear all cache and cookies, then refresh
    Blank Page / Nothing Shows ‚ñº

    Problem

    Browser shows a blank page when you access AEGIS.

    Solutions

    1. Check URL: Ensure you're accessing http://localhost:5050 (not HTTPS)
    2. Enable JavaScript: AEGIS requires JavaScript. Check browser settings
    3. Try Different Browser: Test in Chrome, Firefox, Safari, or Edge. If it works in one, there's a browser compatibility issue
    4. Force Refresh: Press Ctrl+Shift+R (or Cmd+Shift+R on Mac) to do a hard refresh
    5. Check Console: Press F12 to open developer tools, click Console tab, and look for error messages
    Slow Performance / Lag ‚ñº

    Problem

    AEGIS runs slowly or lags when scrolling or filtering.

    Solutions

    1. Disable Info-Level Checkers: Go to Settings ‚Üí Checkers and disable the Info category to speed up analysis
    2. Close Other Applications: AEGIS needs RAM. Close browsers, large documents, and other applications
    3. Reduce Browser Tabs: Having many tabs open consumes memory. Close unused tabs
    4. Restart Server: After many operations, memory usage grows. Restart AEGIS (Ctrl+C, then run app.py again)
    5. Check System RAM: Open Task Manager (Windows) or Activity Monitor (Mac). If AEGIS uses >80% of available RAM, you need more RAM or fewer open applications
    6. Vacuum Database: Over time, the database grows. Run: python scripts/vacuum_db.py to clean it up

    Document Upload & Analysis

    Upload Fails or Gets Stuck ‚ñº

    Problem

    File upload doesn't complete, or stays at 0%.

    Solutions

    1. Check File Size: Default limit is 100MB. If your file is larger, increase in Settings ‚Üí Upload or split the document
    2. Try a Smaller File: Test with a small 1-2 page document. If that works, your original file may be problematic
    3. Check File Format: Ensure the file is in a supported format (.docx, .doc, .pdf, .txt). Don't upload .zip, .exe, or other types
    4. Disable Browser Extensions: Some extensions interfere with uploads. Try incognito/private mode
    5. Check Disk Space: Ensure your hard drive has at least 1GB free space
    6. Network Issues: If on a slow network, upload may timeout. Try wired connection instead of WiFi
    Analysis Produces No Results / Score of 100 ‚ñº

    Problem

    Document analysis completes but shows no issues (score 100) when there should be issues.

    Solutions

    1. Check Enabled Checkers: If you disabled most checkers, few issues will be found. Go to Settings and enable more checkers
    2. Verify File Extraction: Some PDF files or corrupted DOCX files may not extract text properly. Try converting to a different format
    3. Simple Documents: Some very simple, short, or well-written documents genuinely may score 100. This is acceptable
    4. Re-analyze: Delete the scan from Scan History and re-upload the document to force re-analysis
    PDF Not Analyzing Correctly ‚ñº

    Problem

    PDF uploads but analysis produces few or no results.

    Solutions

    1. Check if PDF is Text-Based: AEGIS requires PDFs with extractable text. Scanned PDFs (image-only) cannot be analyzed
    2. Verify PDF Openable: Try opening the PDF in Adobe Reader. If it opens fine, the issue may be with AEGIS's PDF parser
    3. Check PDF Encoding: Some PDFs have unusual encoding. Try exporting from Word as PDF instead of using a scanned copy
    4. Convert to DOCX: As a workaround, convert the PDF to DOCX using an online tool, then upload the DOCX

    Features & Tools

    Roles Not Extracted or Wrong Roles ‚ñº

    Problem

    Role extraction finds few roles, or identifies roles incorrectly.

    Solutions

    1. Review Document Content: Role extraction looks for specific organizational terms. If your document doesn't mention roles explicitly, few will be extracted
    2. Use Adjudication Tool: Manually correct role assignments using the Adjudication tab to train AEGIS
    3. Add Custom Roles: In Settings, add custom role aliases so AEGIS recognizes variations (e.g., "Dev Lead" as "Development Lead")
    Document Compare Not Showing Differences ‚ñº

    Problem

    Compare shows no changes even though you know documents are different.

    Solutions

    1. Verify Different Versions: Ensure you selected two different document versions. If you selected the same document twice, no changes will show
    2. Check Left/Right Selection: Make sure the left is the older version and right is the newer version (order matters)
    3. Subtle Changes: Minor formatting changes (spacing, font) may not be detected. Only content changes show
    4. Refresh Page: If stuck, refresh the page and try comparing again
    Link Validator Shows False Negatives ‚ñº

    Problem

    Link shows as 404 (broken) but works when you click it manually.

    Solutions

    1. Enable Deep Browser Rescan: Some sites require JavaScript execution. Enable this option in Link Validator settings
    2. Check Link Authority: Some sites block automated checkers for security. Manually verify these links
    3. Retry Failed Links: Temporary server issues may cause false failures. Run validation again to confirm
    4. Check Authentication: If links require login, AEGIS can't validate them. Manually verify authenticated links

    Frequently Asked Questions (FAQ)

    • Restrict access to the AEGIS installation folder
    • Back up scan_history.db to encrypted storage
    Q: Can multiple users share AEGIS? ‚ñº
