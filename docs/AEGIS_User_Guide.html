1. Document Review

Automated quality analysis and issue detection

Document Review is the core of AEGIS, providing automated scanning for quality issues, consistency problems, and compliance violations. AEGIS analyzes documents using 88+ quality checkers to identify issues across requirements, grammar, technical content, and compliance areas.

Starting a Review

  1. Click the "Document Review" tile on the dashboard
  2. Select a document from the file browser or drag & drop a new one
  3. Choose a review profile (see below)
  4. Click "Start Review"
  5. Wait for the scan to complete (time varies by document size and profile)
  6. Review results appear in the issues table

Review Profiles

AEGIS offers specialized profiles for different review needs:

Profile Focus Areas Best For Scan Time
Requirements Requirement statements, traceability, compliance Specification documents, SOWs 2-5 min
Grammar Grammar, spelling, style consistency Final review, publication prep 1-3 min
Technical Technical accuracy, terminology, definitions Technical specifications, design docs 3-6 min
Full Scan All checkers enabled (88+ checkers) Comprehensive quality assessment 5-10 min

Understanding Review Results

After a review completes, results display in the Issues Table with the following columns:

Issues Table Columns
  • Issue Type: Category of the issue found
  • Severity: Critical, High, Medium, Low, or Info
  • Location: Page and paragraph number
  • Message: Detailed description of the issue
  • Suggestion: Recommended fix

Severity Levels Explained

Severity Impact Action Required Example
üî¥ Critical Blocking issue Must fix immediately Missing requirements, compliance violations
üü† High Significant issue Should address before approval Undefined terms, missing acceptance criteria
üü° Medium Moderate issue Recommended for fix Inconsistent formatting, vague statements
üîµ Low Minor issue Consider fixing Minor grammar, style suggestions
‚ÑπÔ∏è Info Informational Optional consideration Helpful notes and context

Quality Score & Grade

AEGIS calculates two key metrics for each document:

Metric Range Calculation Interpretation
Quality Score 0-100% 100% - (severity-weighted issues / 10) Higher is better. 90%+ is excellent.
Letter Grade A-F A: 90-100%, B: 80-89%, C: 70-79%, D: 60-69%, F: <60% Traditional grade scale. Aim for A or B.

Filtering Issues

  1. Look for the filter controls above the Issues Table
  2. Click on "Severity" to filter by issue level
  3. Select which severity levels to display
  4. Results update in real-time
  5. Use "Issue Type" filter to focus on specific categories
  6. Combine multiple filters to narrow down results
Filter Best Practices
  • Start with Critical and High severity to address major issues first
  • Don't ignore Low/Info issues‚Äîthey often indicate patterns
  • Export filtered results for stakeholder review

Exporting Results

  1. Click the "Export" button above the Issues Table
  2. Choose format: CSV, PDF, or JSON
  3. Select which columns to include
  4. Choose whether to include all issues or only displayed (filtered) issues
  5. Click "Download"
  6. File downloads to your computer
Export Format Guide
  • CSV: Import into Excel or other tools for analysis
  • PDF: Share with stakeholders, professional presentation
  • JSON: Integration with other systems, programmatic access

2. Statement Forge

Extract, edit, and manage requirement statements

Statement Forge extracts requirement statements from documents and provides comprehensive tools to review, edit, merge, and export them. This tool helps you identify, organize, and manage all requirements within your documents.

Opening Statement Forge

  1. Click the "Statement Forge" tile on the dashboard
  2. Select a document or load a new one
  3. System automatically extracts requirement statements
  4. Statements appear in the Forge interface

Statement Extraction Process

AEGIS uses advanced NLP to identify requirement statements based on:

  • Directive keywords (Shall, Must, Should, May, Will, etc.)
  • Grammatical structures (imperative mood, passive voice)
  • Contextual cues (numbered sections, requirement lists)

Filtering by Directive

Requirement statements are classified by their directive keyword:

Directive Strength Usage Example
Shall Mandatory Binding requirements "The system shall provide authentication"
Must Mandatory Alternative to "shall" "The interface must be user-friendly"
Should Recommended Strongly suggested "Reports should include trend analysis"
May Optional Optional features "Users may customize dashboard themes"
Will Informational Future capabilities "Future releases will support API access"

Managing Statements

Adding Statements

  1. Click "Add Statement" button
  2. Enter the requirement text
  3. Select the appropriate directive from dropdown
  4. Assign category and owner (optional)
  5. Click "Save"

Editing Statements

  1. Click on a statement to select it
  2. Click the "Edit" button or pencil icon
  3. Modify the text or directive as needed
  4. Click "Save" to apply changes

Deleting Statements

  1. Select one or more statements (checkboxes)
  2. Click "Delete" button
  3. Confirm deletion in popup
  4. Statements are removed from the list

Merging Statements

Combine similar or related statements into a single requirement:

  1. Select multiple statements using checkboxes
  2. Click "Merge Statements" button
  3. Review merged text
  4. Confirm the merge
  5. Original statements are replaced with merged version

Splitting Statements

Break compound statements into individual requirements:

  1. Select a statement that contains multiple requirements
  2. Click "Split Statement" button
  3. System suggests split points
  4. Click where you want to split
  5. Confirm to create separate statements

Renumbering Statements

  1. Click "Renumber" button in toolbar
  2. Choose numbering scheme (1, 1.1, REQ-001, etc.)
  3. Set starting number
  4. Click "Apply"
  5. All statements are renumbered automatically

Exporting Statements

  1. Click "Export" button
  2. Choose format: CSV, Excel, or PDF
  3. Select columns to include (Number, Text, Directive, Owner, etc.)
  4. Choose to export all or selected statements
  5. Click "Download"
Statement Forge Best Practices
  • Use consistent directive language across documents
  • Review auto-extracted statements for accuracy
  • Merge redundant or overlapping requirements
  • Export regularly for traceability and archival

7. Link Validator

Validate hyperlinks and URLs in your documents

Link Validator tests all links within documents to identify broken links, redirects, timeouts, and other connectivity issues.

Starting Link Validation

  1. Click "Link Validator" on the dashboard
  2. Choose input method (File Upload or Paste URLs)

Input Methods

File Upload

  1. Click "Choose File" button
  2. Select a document (PDF, DOCX, TXT)
  3. System extracts all URLs automatically
  4. Shows preview of found URLs
  5. Click "Validate"

Paste URLs

  1. Click "Paste URLs" tab
  2. Enter or paste URLs (one per line)
  3. Click "Validate"

Validation Modes

Mode Description Use When
Quick Check Basic HTTP connectivity test Fast check, public URLs only
Full Validation Complete HTTP test with response analysis Thorough check, detect issues
Full + Windows Auth Full validation plus NTLM authentication Internal corporate URLs

Depth Settings

Control link traversal depth:

  • Depth 0: Test provided URLs only
  • Depth 1: Also test all links found on those pages
  • Depth 2: Also test links from linked pages (warning: slow)

Understanding Results

Status Meaning Action Required
‚úì Working Link accessible and responds correctly (HTTP 200) None
‚úó Broken URL not found or not accessible (HTTP 404, 403, etc.) Update or remove link
‚Üí Redirect Link redirects to another URL (HTTP 301, 302) Update to final destination
‚è± Timeout Server didn't respond in time Check URL, may be temporary
üö´ Blocked Access denied by firewall or authentication Check permissions, may be internal

Advanced Settings

  • Timeout: Set response timeout in seconds (default: 30s)
  • Follow Redirects: Continue through redirect chains
  • SSL Verification: Validate HTTPS certificates
  • User Agent: Set browser identification string

Exporting Results

  1. Click "Export Results" button
  2. Choose format (CSV for spreadsheet, JSON for integration)
  3. Click "Download"
Link Validation Cautions
  • Some sites block automated link checking
  • Timeouts may be false positives (network latency)
  • Test at off-peak hours for more reliable results
  • Use Windows Auth mode only for internal URLs

Troubleshooting & Support

Solutions to common issues

Upload Issues

File upload fails or times out
  • Check file size (max 50MB)
  • Try uploading a smaller document first
  • Disable browser extensions that block uploads
  • Try a different browser
  • Check internet connection speed

Review Issues

Review doesn't complete or shows "Unknown Error"
  • Check that document format is supported (PDF, DOCX, TXT, XLSX)
  • Try scanning with "Grammar" profile instead of "Full Scan"
  • Document may be corrupted‚Äîtry re-saving with different tool
  • If file is copy-protected, remove protection before uploading
  • Contact support if error persists
Quality score seems wrong or too high/low
  • Different review profiles produce different scores
  • Document may have been edited since last scan
  • Try re-scanning with same profile for comparison
  • Use Document Compare to see what changed

Statement Forge Issues

No statements extracted or very few found
  • Document may not contain explicit requirement language
  • Requirements may use non-standard directive words
  • Try manually adding statements using "Add Statement" button
  • Check that document language is English

Roles Studio Issues

Roles not loading or graph visualization blank
  • Wait for initial scan to complete (may take a few minutes)
  • Reload page (Ctrl/Cmd + R)
  • Clear browser cache and cookies
  • Try a different browser
  • Check browser console for JavaScript errors

Performance Issues

Application is slow, sluggish, or freezes
  • Close other browser tabs to free memory
  • Clear browser cache (Ctrl/Cmd + Shift + Delete)
  • Disable browser extensions
  • Check available system memory (Task Manager / Activity Monitor)
  • Try with a smaller document set
  • Restart your browser

Data Issues

My data disappeared or seems lost
  • AEGIS maintains scan history (data is not deleted)
  • Check Scan History view to find previous results
  • Filter by document name to locate scans
  • Download from Scan History if needed
  • Data is preserved across browser sessions

Frequently Asked Questions

Quick answers to common questions

General Questions

Q: Is my data stored securely?

A: All document uploads and scans are processed securely on localhost. Data is stored locally on your machine. For cloud deployment options, contact support.

Q: Can I export and archive my scans?

A: Yes! Each scan can be exported as CSV, PDF, or JSON. Use the Export button in Scan History or from individual review results.

Q: Can multiple people use AEGIS simultaneously?

A: AEGIS runs on localhost:5050. Multiple users can access it across a network, but concurrent edits are not simultaneously synchronized. Recommend sequential workflow.

Document Review Questions

Q: What file formats does AEGIS support?

A: PDF, DOCX (Word), TXT (plain text), and XLSX (Excel). Maximum file size is 50MB per document.

Q: How long does a scan take?

A: Depends on document size and profile. Grammar profile: 1-3 min. Full Scan: 5-10 min. Very large documents may take longer.

Q: Can I re-scan a document?

A: Yes! Upload the document again or select it from your scans to re-scan with a different profile. Previous results are preserved in Scan History.

Statement Forge Questions

Q: Can statements be edited after extraction?

A: Yes, fully editable. Modify text, change directive, reassign role, or delete entirely. Changes are saved immediately.

Q: What if a requirement statement wasn't extracted?

A: Manually add it using "Add Statement" button. This is common for non-standard requirement language.

Q: Can I merge similar statements?

A: Yes! Select multiple statements and click "Merge Statements". Useful for consolidating overlapping requirements.

Roles Studio Questions

Q: How are roles identified automatically?

A: AEGIS searches for keywords matching the Role Dictionary (1,056 roles). Any exact match in the document triggers role identification.

Q: Can I add custom roles?

A: Yes! Use the Role Dictionary tab to add new roles to the library. They will be recognized in future scans.

Q: What does "Adjudication" mean?

A: Review and classify each role as Confirmed (correct), Deliverable (ready), or Rejected (not relevant). Helps ensure quality of role analysis.

Q: Why is the RACI matrix empty?

A: Complete Adjudication first to confirm which roles are valid. Only confirmed roles appear in RACI analysis.

Metrics Questions

Q: What's the difference between Quality Score and Grade?

A: Quality Score is the percentage (0-100%). Grade is the letter equivalent (A=90-100%, B=80-89%, etc.). They show the same information differently.

Q: Can I compare metrics between documents?

A: Yes! Use Metrics & Analytics tab "Documents" sub-tab to see detailed stats for each document. Use Portfolio to see all documents side-by-side.

Advanced Questions

Q: How do I integrate AEGIS with other tools?

A: AEGIS exports to CSV and JSON formats compatible with Excel, databases, and programming tools. Contact support for custom API integration.

Q: Can I automate scanning and reporting?

A: Currently, AEGIS requires manual scanning from the UI. Batch processing and scheduling are planned for future releases.

Q: Is there a command-line or API interface?

A: Not in v5.0.0. AEGIS is a web application accessible via browser at localhost:5050. API and CLI are on the roadmap for future versions.
